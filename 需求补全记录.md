## 一、系统定性

- **系统名称**：客户服务中心（需开发）
- **技术方向**：低代码 + 定制后端混合架构（见下方说明）
- **通知渠道**：微信服务号（现有服务号），无需对接企业微信

**技术选型建议：**
- **定制后端服务**（Node.js 或 Python/FastAPI）：负责所有外部 API 中转（右豹APP、飞书、第三方导师系统、微信服务号）、SLA 定时任务、消息通知队列与重试机制
- **数据库**：MySQL 主库 + Redis 缓存层（缓存右豹/第三方API同步数据，每小时定时刷新，支持手动触发更新）
- **管理后台前端**：可采用 Vue3 + 开源后台模板（如 Arco Design Pro）快速搭建 CRUD 管理页面；审核工作台、通知失败队列等复杂页面定制开发
- **消息队列**：处理通知发送的异步重试（服务号失败 → 记录失败队列）

**系统菜单导航结构：**

```
📊 Dashboard（数据看板）

📋 用户管理
  └─ 用户主档表
  └─ 付费记录管理

✅ 入群审核
  └─ 入群审核工作台
  └─ 消息通知记录（只读）
  └─ SLA 预警记录（只读）

👥 飞书运营
  └─ 飞书好友优先级管理

🏢 组织管理
  └─ 客服管理
  └─ 导师管理
  └─ 门派管理

⚙️ 系统设置
  └─ 权限与账号管理
```

### 模块 1：用户数据录入与审核

**非付费用户流程：**
1. 用户扫码添加右豹客服企业微信
2. 客服分流：意向付费 → 转专属顾问；无意向 → 进入数据录入
3. 客服在系统"用户主档表"中直接录入用户数据（字段：右豹编码、飞书ID、飞书手机号、飞书昵称）；录入无需用户自填，客服直接操作
4. 数据录入完成后，客服向用户发送对应飞书群二维码
5. 用户申请进群时须备注"右豹编码 + 飞书注册手机号"
6. 系统通过飞书API获取进群申请信息，生成入群审核表记录，审核员在系统内审核通过/拒绝

**飞书 API 不可用时的降级方案：**
- 管理员可在后台手动导入入群申请数据（替代飞书API自动拉取）
- 管理员在系统内主动操作审核（通过/拒绝），SLA 以手动录入记录的申请时间为计时起点

**审核操作交互（审核工作台）：**
- 审核员在入群审核列表页直接操作，无需进入详情页
- 列表展示字段：右豹编码、飞书昵称、近10天关键词申请数、近10天订单数、申请时间、申请状态（待审核橙色/已超时红色）、操作按钮
- **[通过]**：一键操作 → 系统自动更新状态为已通过 → 调飞书API批准入群 → 发送微信服务号通知「恭喜您，您的入群「{群名称}」审核已通过」
- **[拒绝]**：弹窗要求填写拒绝原因 → 确认提交 → 更新状态为已拒绝 → 发送微信服务号通知（含拒绝原因 + H5修改指引）→ 若服务号推送失败则进入通知失败记录
- 详情页展示入群审核表全部字段

**飞书入群状态同步（审核通过后）：**
- 系统审核通过后自动调飞书 API 批准入群
- 若飞书侧先批准入群，系统通过以下三种方式同步状态：①定时轮询；②飞书 webhook 回调；③管理员后台手动触发刷新

**重复提交规则：**
- 审核进行中：不支持重复提交，系统拦截
- 审核被拒后：用户可重新提交，作为全新入群审核记录，SLA 重新计时；旧记录自动归档

**付费用户流程：**
1. 协助用户完成右豹APP注册
2. 客服在系统"用户主档表"录入用户数据；要求用户提供右豹编码和右豹ID的截图（共2张）供客服核对
3. 客服根据截图核对录入内容是否正确
   - **核对无误：** 继续后续步骤
   - **核对有误：** 请用户重新提供截图；核对错误记录留存系统，方便溯源
4. 核对无误后，在飞书"第三职业"交付体系中添加用户信息
5. 对接客服补充完善用户的飞书注册手机号及飞书ID

### 模块 2：录入付费用户

**功能描述：** 由负责人将线下付费行为录入系统，触发用户付费状态变更

**录入方式：**
- 支持 Excel 批量导入
- 支持单条手动新增

**录入字段：** 右豹编码、飞书手机号、付费金额、付费时间、付费对接人

**校验逻辑：**
- 通过 API 校验右豹编码在右豹APP中是否真实存在
- 批量导入时校验规则如下（两类失败均计入失败条目，不可导入）：
  - **同批次重复右豹编码：** 对重复的编码条目报错标记为失败；同批次中未重复的条目不受影响，仍可正常导入
  - **其他失败条目（如编码不存在、已为付费学员等）：** 该条目标记为失败，跳过不导入
- 该右豹编码对应用户已为付费学员 → 报错提示，不支持重复录入
- 付费金额为负数 → 拦截报错；付费金额为 0 → 不做拦截，允许通过
- 付费时间格式：仅需精确到年月日（YYYY-MM-DD）
- 校验通过后：自动将用户表"是否付费学员"字段更新为已付费
- 导入完成后展示汇总提示：「成功 X 条 / 失败 X 条」
- 支持导出失败数据明细为 Excel，供操作人员核查

**修改与删除：**
- 支持修改单条付费记录中的：付费金额、付费时间、付费对接人
- 支持删除付费记录；删除后对应用户"是否付费学员"自动回退为"未付费"
- 修改和删除均记录操作日志（含操作人、操作时间、变更前后内容）

**API 不可用时：** 右豹APP API 不可用时，本模块批量导入及单条录入的右豹编码校验全部失败，整批/单条均无法提交，需等待 API 恢复后重试

**操作记录：** 每条录入数据存入"录入付费用户"菜单，作为操作日志留存

### 模块 3：消息通知

**通知渠道：** 微信服务号（主）/ 短信（降级）

**触发场景：**

| 场景 | 通知对象 | 通知内容 |
|---|---|---|
| 入群申请审核通过 | 用户 | 「恭喜您，您的入群「{群名称}」审核已通过」 |
| 入群申请被拒 | 用户 | 拒绝原因 + H5 修改指引页链接 |
| 审核超时（>3天） | 审核员 | 系统自动发送加急提醒（微信服务号） |
| 审核超时二次催促 | 审核员 | 首次提醒后8小时内仍未处理，再发一次 |

**OpenID 绑定机制：** 微信服务号配置「达人绑定」H5菜单，用户主动在菜单内填写右豹编码完成 OpenID 与右豹编码的关联；OpenID 存储于数据库，不在后台管理界面展示

**拒绝原因机制：** 审核人员在拒绝时自定义填写拒绝原因，内容原样推送至 H5 页面；若未填写，H5 页面展示默认文案：「审核未通过，详细原因请联系客服」

**通知失败降级：** 微信服务号通知发送失败时（如用户未完成 OpenID 绑定），系统自动判断推送失败，记录至**通知失败队列**；客服可在后台通知失败记录菜单中查看，通过企微/飞书线下联系用户；短信降级需人工执行

**通知失败记录菜单（只读）：**
- 记录字段：通知时间、右豹编码、通知场景、通知渠道、推送状态（成功/失败）
- 客服角色可见，供线下跟进使用

**SLA 预警记录菜单（只读）：**
- 记录字段：预警时间、入群审核记录ID、预警类型（首次超时/二次催促）、发送状态
- 管理员/审核员角色可见

**短信内容格式：** 「您在【{群名称}】的入群申请已{通过/驳回}，详情请点击：{H5链接}」；人工发送时须规范附上 H5 链接及群名称

### 模块 4：SLA 超时预警

- **时效标准：** 3 自然日内正常处理
- **超时时间定义：** 申请提交当天计为第1天，第3个自然日的 **23:59（北京时间，UTC+8）** 为截止时间；申请时间的时间精度（秒/分钟级）不影响超时判定，以自然日边界为准
- **计时起点：** 入群审核表记录的"申请时间"字段开始计算
- **超时触发：** 超过 3 自然日（即第3个自然日 23:59 后仍未处理），系统自动向**审核员**（账号标记为"是否为审核员"的管理员）发送第一次加急提醒（微信服务号）
- **二次催促：** 首次超时提醒发出后 8 小时内，若仍未处理，再发送一次催促；此后不再重复催促
- **旧记录处置：** 审核被拒后用户重新提交，原已超时/已拒绝记录自动归档，不再触发新的超时提醒；新提交记录重新独立计时
- **定时任务过滤条件：** 只扫描**申请状态 = 待审核**的记录；已归档、已拒绝、已通过的记录不参与超时计时，不触发预警
- **执行主体：** 系统自动触发，无需人工干预

### 模块 5：飞书好友添加优先级管理

**功能描述：** 展示待入群/入群审核中的用户优先级列表，支持一键发起飞书好友申请并跟踪添加状态

**用户范围：** 仅展示**申请状态为待审核或入群审核中**的用户，不展示全量用户主档表

**优先级规则：**
- **优先添加：** 近10天内有"关键词申请"或"信息回填"动作的用户
- **暂缓添加：** 近10天内无任何申请或关键词动作的用户

**一键发起飞书添加：**
- 系统支持以**具体客服/导师账号**的身份向用户发起飞书好友申请（调用飞书 API）
- 前提：客服/导师须在系统内完成飞书账号 OAuth 授权；账号离职/停用时需及时撤销授权，否则仍可能以其账号发起申请
- 发起申请后，飞书API回调通知系统对方接受/拒绝状态，自动更新飞书添加记录表中的申请状态
- 也支持操作人手动标记"我已添加"

**添加状态跟踪：**
- 状态字段存于飞书添加记录表（见数据结构章节），仅在本模块视图展示，不写入用户主档表
- 申请状态枚举：待接受 / 已接受 / 已拒绝 / 超时
- 视图支持按添加状态筛选，便于负责人监控"今日哪些用户尚未完成好友添加"

**数据同步：**
- 近10天动作数据异步预加载，每天定时更新（非实时）
- **API 不可用时：** 展示上次同步的历史数据，页面顶部提示「数据暂未更新，请稍后刷新」

### 模块 6：权限控制

**控制维度：**
- **菜单权限：** 不同角色可见/可访问的菜单项不同
- **操作权限：** 对同一菜单下的新增、修改、删除、导入、导出等操作按角色分配
- **字段显隐：** 敏感字段（如付费金额、付费对接人等）按角色控制显示与隐藏

**角色与账号管理：**

角色体系由系统管理员在后台自行配置，流程如下：
1. **配置角色组：** 管理员创建角色组并为其分配菜单/操作/字段权限
2. **新增管理员账号：** 创建后台操作人员账号时需填写以下字段：
   - 管理员名称
   - 登录账号
   - 初始密码
   - 所属角色组（必选）
   - 是否为审核员（布尔值；标记为审核员后，该账号可收到模块3/4的入群审核相关通知）
3. **账号自助修改：** 管理员登录后台后可自行修改昵称和登录密码

**审核员说明：** 审核员不是固定角色名，而是管理员账号上的一个属性标记；负责飞书入群审核的账号需勾选"是否为审核员"，SLA 超时通知和审核驳回通知均发送给所有被标记为审核员的账号

**操作日志：** 所有涉及数据变更的操作（录入、修改、删除、审核通过/拒绝）均需记录操作日志，包含：操作人、操作时间、操作类型、操作前后数据内容

> ⚠️ **待补充：** 具体角色清单及各角色对应的菜单/操作/字段权限矩阵，待 PRD 阶段统一设计

### 模块 8：导师管理
