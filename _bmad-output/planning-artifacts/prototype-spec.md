### 1.3 提示组件使用规范

#### Toast（轻提示）
- **使用场景：** 操作完成的即时反馈；用户在线时的系统通知；403 权限变更提示
- **展示方式：** 右上角弹出，3 秒自动消失，可点击跳转
- **不适用：** 需要用户决策的场景、持续性状态

#### Banner（持续性通知条）
- **颜色：** 告警级使用 `badge-orange` 样式；信息级使用灰色
- **展示方式：** 操作区域顶部固定，**必须使用 sticky 定位**
- **可关闭性：** 告警级不可关闭（持续至状态恢复）；信息级可关闭

#### Inline（内联提示）
- **使用场景：** 表单字段的实时校验反馈
- **展示方式：** 紧贴对应输入框下方

#### Modal（弹窗）
- **使用场景：** 需要用户立即决策的一次性操作（确认拒绝、导入结果、删除确认、重置密码）
- **展示方式：** 居中遮罩，用户必须操作后才可继续

#### Drawer（抽屉）
- **使用场景：** 内容区域较大的配置操作（权限分配）
- **展示方式：** 从右侧滑入，宽度 800px，背景半透明遮罩，顶部 sticky 按钮条

#### 同页面多提示共存优先级

```
Modal / Drawer（最高，遮罩所有内容）
  └── Banner（粘性顶部，不遮挡内容区）
        └── Toast（右上角浮层）
              └── Inline（内联，随表单存在）
```

#### 全局 HTTP 状态码处理

| 状态码 | 场景 | 前端处理 |
|---|---|---|
| **401** | Token 失效 / 会话过期 | 全屏 Modal："登录已过期，请重新登录" + [去登录] 按钮 |
| **403** | 有效 Token，但无权限（权限已被变更） | Toast："您的权限已更新，请刷新页面重试" |

> `[技术标注]` 后端必须严格区分 401（认证失败）和 403（权限不足），前端全局拦截层分别处理，不可混用同一提示文案。

### 1.4 标准页面四态

| 状态 | 定义 | 视觉要求 |
|---|---|---|
| **空态** | 无任何数据 | 居中图标 + 主文案 + 引导操作按钮；筛选结果为空时提示"未找到符合条件的记录" |
| **加载态** | 数据请求中 | 列表区域骨架屏（Skeleton），避免全屏 loading |
| **局部异步态** | 部分数据可用，部分同步中 | 同步中字段显示 `⏳ 同步中`（badge-blue） |
| **批量操作态** | 勾选多条记录后 | 列表顶部激活操作栏，显示"已选 X 条"；取消所有勾选时收起 |

#### 列表分页策略规范

> 适用于所有列表型页面，按场景分类执行，不可省略。

| 列表类型 | 分页方式 | 每页条数 | 备注 |
|---|---|---|---|
| 主列表（用户主档 / 付费记录 / 通知失败记录 / SLA预警 / 组织管理） | 传统分页，含页码 + 跳页输入框 | 20条 | 支持点击列头切换排序字段 |
| 入群审核工作台 | 传统分页，含页码 | 50条 | 高频审核场景，每页更多减少翻页次数 |
| Dashboard 概览表格（客服/导师/门派） | 不分页，全量展示 | 上限50条 | 超出50条时表格底部提示"显示前50条，[查看完整列表]"跳转对应管理页 |
| 详情页操作日志 | 默认折叠，展开后滚动加载 | 默认展示最近20条 | 点击 [加载更多] 追加20条，直至全部加载完毕 |

### 1.5 系统内通知规范

| 事件 | 用户在线时 | 用户离线后登录 |
|---|---|---|
| 批量导入 API 校验完成 | 右上角 Toast（含跳转链接） | 顶部 Banner（可关闭） |
| 数据导出文件生成完成 | 右上角 Toast（触发浏览器自动下载 + 含 24 小时有效的下载链接） | 顶部 Banner（含 24 小时有效的下载链接，可关闭） |
| SLA 超时预警（系统发出） | 右上角 Toast | 顶部 Banner（可关闭） |
| 外部 API 恢复正常 | 右上角 Toast（降级 Banner 自动消失） | — |

> **导出文件 24 小时有效期：** 用户关闭页面或错过自动下载时，可在通知中心找到链接重新下载；超过 24 小时后链接失效，需重新导出。

### 1.6 导入记录功能规范

> **「导入记录」是存在批量导入功能的各管理页面内的子功能 Tab，不是独立页面。**
> **Tab 可见性与"导入"操作权限绑定**：有批量导入权限的角色可见；无权限角色不展示 Tab。`[字段显隐由角色权限配置决定]`

#### 导入记录 Tab 触发时机
- 执行批量导入完成后，页面自动切换至「导入记录」Tab
- 用户可手动点击 Tab 切换

#### 批次列表字段

| 字段 | 说明 |
|---|---|
| 导入批次号 | 全局自增，格式 `IMP-000001`（6 位，全局唯一） |
| 导入时间 | 管理员触发导入的时刻 |
| 导入人 | 操作人账号名称 |
| 文件名 | 原始上传文件名 |
| 上传总条数 | 文件内原始记录数 |
| 本地校验通过 | 通过格式/重复校验的条数 |
| API 校验通过 | 最终入库成功条数 |
| 失败条数 | 两阶段合计失败数 |
| 校验状态 | 进行中 / 已完成 / 部分失败 |
| 操作 | [下载异常 Excel]（失败条数 > 0 时可用） |

#### 进度自动刷新
- **5 秒自动轮询**，仅在 Tab 激活且存在"进行中"批次时启动
- 所有批次完成后自动停止；**用户切换 Tab 或离开页面时立即停止**
- `[技术标注]` 通过 Page Visibility API 或组件卸载钩子实现
- 底部展示"最后刷新：X 秒前"+ [立即刷新] 手动按钮

### 1.9 数据导出规范

**导出入口：** 各模块列表页顶部操作栏，[导出] 按钮（权限：`[按角色权限配置决定，导出权限独立控制]`）

**适用模块：**

| 模块 | 导出范围 | 导出格式 | 文件名 |
|---|---|---|---|
| 用户主档 | 当前筛选结果（无筛选时全量） | Excel（.xlsx） | `用户主档_导出_YYYY-MM-DD_HH-mm.xlsx` |
| 付费记录 | 当前筛选结果 | Excel（.xlsx） | `付费记录_导出_YYYY-MM-DD_HH-mm.xlsx` |
| 入群审核工作台 | 当前筛选结果 | Excel（.xlsx） | `入群审核_导出_YYYY-MM-DD_HH-mm.xlsx` |

**导出交互规则：**
- 点击 [导出] → 后台异步生成文件 → Toast 提示"正在生成导出文件，完成后将自动下载"
- 生成完成 → 浏览器自动下载 + 系统通知内保留 24 小时有效下载链接（见 1.5 节）
- 数据量 > 1000 条时：提示"导出数量较多，预计需要 X 秒，请稍候"
- 导出过程中 [导出] 按钮禁用（loading 态），防止重复触发

### 2.1 入群审核工作台（P0，最高优先级）

#### 入群审核状态完整 Badge 对照表

| 状态 | Badge Class | 文案 | 触发条件 |
|---|---|---|---|
| 待审核（0~2日） | `badge-orange` | `待审核` | 申请已创建，等待处理 |
| 即将超时（2.5日+） | `badge-orange` + ⏰ | `即将超时` | 等待时长超过2.5自然日 |
| 已超时（3日+） | `badge-red` | `已超时` | 等待时长超过3自然日 |
| 处理中 | `badge-gray` | `处理中（预计1分钟内完成）` | 已调用飞书API，等待回调 |
| 已通过 | `badge-green` | `已通过` | 飞书回调确认入群成功 |
| 已拒绝 | `badge-red` | `已拒绝` | 审核员执行拒绝操作 |
| 已归档 | `badge-gray` | `已归档` | 重新提交/超30日/手动归档触发 |

#### 列表页字段定义

| 字段 | 说明 | 宽度权重 |
|---|---|---|
| 状态标记 | 使用上方状态 Badge 对照表对应样式 | 固定宽，最左 |
| 申请人飞书昵称 | 主要标识 | 中等 |
| 右豹编码 | 关键业务字段 | 固定宽 |
| 申请时间 | 带来源标注（`飞书同步` / `手动导入` 小标签） | 中等 |
| 等待时长 | `已等待 X 天 X 小时`；超 3 日变红 | 固定宽 |
| 近 10 天动作数 | 关键词申请数 + 订单数（来自右豹 API） | 中等 |
| 操作列 | 一键通过 / 拒绝（审核员权限）；普通客服只读 | 固定宽，最右 |

**列表行交互：** 点击行进入审核详情页；操作列按钮独立触发对应操作。

#### 筛选维度
- 状态（超时 / 待审核 / 已通过 / 已拒绝 / 已归档）
- 申请时间范围
- 处理人

#### 已归档状态说明
- **触发一（立即归档）：** 用户被拒后重新提交时，旧记录**立即**变为"已归档"，新记录同时创建，不共存于待审核视图
- **触发二（自动归档）：** 被拒后超过 **30 自然日**未重新提交，系统第 31 日零点批量归档。操作日志：操作人="系统自动"，操作类型="自动归档（超过30日未重新提交）"
- **触发三（手动归档）：** 管理员在审核详情页手动归档，需填写归档原因
- 已归档记录**默认不展示**，需在筛选器主动选择"已归档"才可查看

#### 审核通过操作规格
- **单条通过：** 直接执行，**无需确认弹窗**（高频操作）
- 操作后：列表行立即更新为"处理中"（badge-gray），待飞书 API 回调后更新为"已通过"（badge-green）
- **飞书 API 调用失败：** Toast 提示"飞书入群操作失败，请稍后重试或手动处理"，列表行状态回滚至"待审核"
- **批量通过：** 无确认弹窗，点击后直接执行，Toast 提示"已提交 X 条通过操作，处理结果将逐步更新"

#### 单条拒绝弹窗规格
- 标题："确认拒绝入群申请"
- 内容：展示申请人姓名 + 右豹编码
- 拒绝原因：多行文本框，最大 200 字，留空提示"将使用默认拒绝文案"
- 操作：[取消] [确认拒绝]
- 确认后：弹窗关闭，列表行更新为"已拒绝"，Toast 提示"操作成功"

#### 批量拒绝弹窗规格
- 标题："批量拒绝入群申请"
- 内容区：展示被选中的所有申请人列表（飞书昵称 + 右豹编码），可滚动
- 数量限制：单次最多 **1000 条**；超出时提示"已达单次操作上限（1000 条），请分批处理"
- 拒绝原因：多行文本框，最大 200 字，统一适用所有被选条目；留空则用默认文案
- 确认按钮文案：**"确认拒绝 X 条"**
- 确认后：所有被选行更新为"已拒绝"，Toast 提示"已拒绝 X 条申请"，勾选状态清除

#### 空态
- 无任何申请：图标 + 文案"暂无待审核申请"
- 筛选结果为空：文案"未找到符合条件的申请记录"+ [清除筛选] 按钮

#### 手动同步飞书入群状态（FR19）
- **入口：** 列表页顶部操作栏 [手动同步飞书状态] 按钮（管理员权限）
- **触发范围：** 同步所有状态为"处理中"的审核记录（已调用飞书 API 但尚未收到回调的记录）
- **触发后：** 按钮立即进入 loading 态（禁用，防止重复触发），Toast 提示"正在同步飞书入群状态，请稍候"
- **同步完成：** 按钮恢复可点击，Toast 提示"同步完成，已更新 X 条记录状态"

#### 「处理中」状态超时补偿机制（FR19 Webhook 兜底）
- 审核操作触发后，列表行显示"处理中"（badge-gray），提示文案：`处理中（预计 1 分钟内完成）`
- 若 **30 分钟**内未收到飞书 webhook 回调，系统自动触发补偿性轮询查询实际入群状态
- 补偿查询后状态更新为"已通过"或"操作失败"；若飞书 API 此时也不可用，状态更新提示文案改为：`处理中（飞书服务异常，请稍后手动同步）`
- `[技术标注]` 30 分钟补偿轮询由后端定时任务实现，前端无需额外处理

#### 重复提交拦截（FR20）
- 飞书 API 端拦截同一用户的重复入群申请，系统对重复申请**静默丢弃**，不在审核工作台创建重复记录
- 审核员无感知，原"待审核"记录保持可见；原型中无需为此设计特殊 UI

#### 手动导入入口（飞书 API 降级时）
- 顶部告警 Banner 提供 [前往手动导入] 按钮
- 导入完成后，页面内展示「导入记录」Tab（规格见 1.6 节）

#### 导出入口
- 列表页顶部 [导出] 按钮，导出当前筛选结果，规格见 1.9 节

### 2.2 用户主档列表 + 详情页（P0）

#### 列表页字段定义

| 字段 | 说明 |

| 右豹编码 | 全局唯一主键 |
| 飞书昵称 | 主要显示名 |
| 是否付费 | badge-green"付费学员" / badge-gray"普通用户" |
| 所属客服 | 关联展示 |
| 所属导师 | 关联展示 |
| 所属门派 | 关联展示 |
| 录入时间 | 排序默认按此倒序 |

#### 「已拒绝」好友申请处理规格

- 好友申请状态变更为"已拒绝"时，该记录**自动移出主列表**，进入页面底部「需人工决策」折叠分区
- 「需人工决策」分区字段同主列表，额外展示"被拒绝时间"和"被拒绝次数"
- 操作选项：
  - **[重新申请]**：弹出确认 Modal，标题"确认重新发起好友申请"，内容"该用户曾拒绝好友申请，确认重新向对方发起？"，操作 [取消] [确认发起]；确认后记录重置为"未申请"状态，回到主列表
  - **[标记放弃]**：直接执行（无确认弹窗），记录从「需人工决策」分区移除，进入已归档态

#### 超时规则与视图策略
- 好友申请发出后 **7 自然日**无响应，状态自动变更为 `超时`（badge-gray）
- 主视图**默认排除"超时"状态**（仅展示：未申请 / 待接受 / 已接受 / 已拒绝）
- 筛选器提供"超时"选项；顶部计数提示（数量 > 0 时）："当前有 X 条申请已超时，[查看]"
- 超时后可重新发起申请（记录重置为"未申请"状态）

#### 飞书 OAuth 授权状态与完整流程

在发起申请前，客服/导师账号需完成飞书 OAuth 授权。授权状态在列表上方展示：

| 授权状态 | 展示方式 | 操作 |
|---|---|---|
| 未授权 | 黄色提示条 | [立即授权] → 新标签页打开飞书 OAuth 页面 |
| 授权中 | 灰色提示条 | 不可操作 |
| 授权成功 | 不展示提示条 | — |
| 授权失败 | 红色提示条 | [重新授权] |
| 授权过期 | 橙色提示条 | [重新授权] |

**OAuth 完整回调流程：**

```
[触发] 点击 [立即授权] / [重新授权]
  ↓ 在新标签页打开飞书 OAuth 页面
  ↓
[飞书页面操作]
  ├── 用户确认授权 → 飞书回调系统 URL
  │     ├── 回调成功 → 授权状态更新为"授权成功"，原页面提示条自动消失
  │     └── 回调失败 → 授权状态更新为"授权失败"，展示失败提示条
  │
  └── 用户取消/关闭窗口 → 授权状态保持原状，不变更
```

> `[技术标注]` 原页面通过 **`BroadcastChannel API`** 跨标签页监听授权结果，授权完成后自动刷新状态，无需用户手动操作。`BroadcastChannel` 专为同源跨标签页通信设计，不暴露于全局存储（替换原 `localStorage` 事件方案，避免 storage 事件的安全顾虑）。

#### 「以客服/导师身份申请」操作交互

点击 [以客服身份申请] 后弹出账号选择弹窗：

| 字段 | 说明 |
|---|---|
| 账号名称 | 客服/导师姓名 |
| 飞书授权状态 | 已授权（可选）/ 未授权（灰色置底，不可选） |
| 本月发出申请数 | 当月以该账号身份发出的好友申请总数 |

- 若无已授权账号：提示"暂无已授权的客服账号，请先完成飞书授权"
- 确认后：弹窗关闭，列表行状态 → `待接受`，Toast 提示"已以 [账号名] 的身份发起申请"
- 点击 [以导师身份申请]：同上，展示已授权的导师账号列表

#### 申请状态流转

| 操作结果 | 列表行状态 |
|---|---|
| 申请发出成功 | → `待接受`（badge-orange） |
| 飞书 API 回调：已接受 | → `已接受`（badge-green） |
| 飞书 API 回调：已拒绝 | → `已拒绝`（badge-red） |
| 超过 7 自然日无响应 | → `超时`（badge-gray） |

- **删除 vs 禁用区别：** 禁用=账号保留可恢复；删除=账号永久移除，日志保留

## 05 业务规则边界（已确认）

### SLA 计时规则

> **本文档规则覆盖 PRD FR15：**

| 场景 | 计时起点 | 节假日 |
|---|---|---|
| 飞书 API 正常同步 | 飞书申请提交时间（API 返回字段） | 自然日计算，节假日照算 |
| 飞书 API 降级·手动导入 | 系统导入操作时间（管理员触发时刻） | 自然日计算，节假日照算 |

> 截止时间：第 3 日 23:59（UTC+8）为 SLA 边界。

### 重复编码处理策略

| 场景 | 处理方式 |
|---|---|
| 单条录入时编码重复 | Inline 提示，阻断提交，提供跳转已有记录链接 |
| 批量导入时编码重复 | 报错跳过，写入异常 Excel |
| 批量导入时 API 校验发现编码无效 | 报错跳过，Badge 显示"❌ 编码无效" |

### 补校验机制
- 管理员跳过校验录入的记录进入 `⚠ 编码待验证` 状态
- API 恢复后 30 分钟内自动触发后台重新校验
- 校验通过 → ✅ 已验证；失败 → ❌ 验证失败，需人工处理

### 付费状态变更规则
- 付费状态只能通过付费记录的新增/删除操作触发
- 原型中"是否付费"字段为只读展示

### 入群审核归档规则

| 触发类型 | 时机 | 操作日志 |
|---|---|---|
| 重新提交立即归档 | 用户被拒后重新提交申请的瞬间 | 操作人=实际触发人 |
| 超期自动归档 | 被拒绝后第 31 日零点 | 操作人="系统自动" |
| 手动归档 | 管理员主动操作 | 操作人=管理员，需填写原因 |

### 飞书好友申请超时规则
- 发出后 **7 自然日**无响应自动标记为超时
- 超时记录默认隐藏，筛选器可查；顶部计数提示
- 超时后可重新发起申请

### 统计口径规则
- **"建联"定义：** 客服已将该用户录入用户主档（存在主档记录）
- **漏斗统计口径：** 以时间范围内的新增量为准，不是累计存量
- **删除不影响历史统计：** 以操作发生时间为准，事后删除记录不回溯漏斗/指标数据

### 导师门派关系维护规则
- 导师归属门派只能在导师管理侧操作
- 门派详情页导师列表标注为 `只读，从导师管理维护`

### 操作日志分布规则

各业务表日志分散在各自页面详情页底部展示，无统一日志入口：

| 业务表 | 操作日志记录字段 |
|---|---|
| 用户主档 | 操作时间、操作人、操作类型、变更字段、变更前值、变更后值 |
| 付费记录 | 操作时间、操作人、操作类型（新增/修改/删除）、变更字段、变更前值、变更后值 |
| 入群审核 | 操作时间、操作人、操作类型（通过/拒绝/归档/手动导入）、拒绝/归档原因（如有） |
| 客服 | 操作时间、操作人、操作类型、变更字段、变更前值、变更后值 |
| 导师 | 操作时间、操作人、操作类型、变更字段、变更前值、变更后值 |
| 门派 | 操作时间、操作人、操作类型、变更字段、变更前值、变更后值 |
