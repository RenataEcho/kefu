# 产品简报：客户服务中心

## 执行摘要

客户服务中心是一套为右豹平台内部运营团队专属打造的客服管理系统。它将当前分散在企业微信、飞书、Excel 等多个工具中的客服工作流整合为一个统一的数字化后台，打通用户数据录入、入群审核、消息通知、SLA 预警、飞书好友添加等全链路操作。

系统面向客服人员、导师和管理员三类核心角色，覆盖从用户初次接触到完成交付的完整生命周期。核心价值在于：用一个系统替代多工具切换，用结构化数据替代人工记录，用自动化流程替代重复人工操作，从而显著提升客服效率，降低数据录入错误率。

右豹平台目前处于规模扩张阶段，客服团队和导师体系持续壮大。现有的分散工具组合已无法支撑业务增长，信息孤岛、手工操作带来的数据错误、沟通延迟和工作积压正在成为实质性瓶颈。客户服务中心是打通这一瓶颈的关键基础设施投入。

---

## 核心问题

右豹平台客服团队当前面临四类核心挑战：

**流程碎片化**：用户从咨询到入群的完整流程横跨企业微信、右豹APP、飞书、Excel"数据录入库"多个工具，客服需在多个界面手动切换，信息无法自动流转，单次服务操作步骤冗长。

**数据准确率低**：用户关键数据（右豹编码、飞书手机号、飞书ID等）靠人工录入 Excel，无实时校验，重复录入、字段缺失、编码填错时有发生，事后补救成本高。

**审核工作被动低效**：入群审核依赖人工在飞书群内手动核对申请备注与 Excel 记录，无 SLA 管控，超时无预警，审核员无统一工作台，处理效率低且存在合规风险。

**管理数据断层**：客服、导师、门派的业务数据（建联用户数、学员数、收益等）分散于不同工具，管理层无法实时掌握全局运营状况，更无法下钻分析。

---

## 解决方案

客户服务中心以**低代码 + 定制后端混合架构**构建，提供统一的 Web 管理后台，覆盖以下核心功能模块：

| 模块 | 核心能力 |
|---|---|
| **用户数据管理** | 客服直接在系统录入用户主档，右豹APP API 实时校验编码真实性，付费记录支持批量导入与全操作日志 |
| **入群审核工作台** | 飞书进群申请自动同步，一键通过/拒绝，3自然日 SLA 自动预警，超时红色标记 |
| **消息通知** | 审核结果自动通过微信服务号推送，失败自动记录待处理队列，SLA 超时自动推送审核员 |
| **飞书好友添加** | 按优先级排序待添加用户，以指定客服/导师身份一键发起飞书申请并自动追踪状态 |
| **组织管理** | 统一管理客服、导师、门派，数据关联自动汇总，第三方 API 自动同步导师学员绑定关系 |
| **Dashboard 数据看板** | 按角色差异化展示核心运营指标（管理员看全局，客服看个人），内含咨询→录入→入群→付费全程转化漏斗，支持日期范围筛选 |
| **权限控制** | 菜单/操作/字段三维度精细权限，审核员属性标记，完整操作日志体系 |

---

## 核心差异点

**深度定制，非通用 CRM**：系统完全针对右豹平台运营场景设计，流程、字段、规则均来自真实业务，无冗余功能。

**四路外部 API 原生集成**：右豹APP、飞书、第三方导师系统、微信服务号四路 API 打通，数据自动流转。每路 API 均有明确降级策略，外部服务故障时核心业务可降级运行，不中断。

**合规可追溯**：所有数据变更（录入、修改、删除、审核）均记录操作日志，含操作前后数据快照，满足内部审计和问题溯源需求。

**内部工具执行壁垒**：与右豹平台多系统的深度集成、与客服团队实际工作流高度贴合的交互设计，以及完整的审计链——这些无法被通用 SaaS 工具简单替代。

---

## 服务对象

**主要用户 — 客服人员（普通客服 / 专属顾问 / 进群审核客服）**
每日处理大量用户接入、数据录入、入群审核工作，当前工具分散且操作繁琐。核心需求：更快完成单个工单、减少因数据错误导致的返工、清楚知道哪些任务待处理。

**次要用户 — 导师**
负责学员带教，需了解自己的学员情况与收益数据。系统通过第三方 API 自动同步导师学员绑定关系，导师无需手动维护数据。

**管理层 / 系统管理员**
需要全局视角了解业务运转状况，管理团队账号和权限。Dashboard 和组织管理模块专为此角色设计。

---

## 成功衡量指标

1. **效率提升**：客服完成单用户完整录入+审核的平均时长下降 ≥ 30%
2. **数据准确率**：用户主档关键字段（右豹编码、飞书手机号）录入准确率 ≥ 99%
3. **SLA 达标率**：入群审核 3 自然日内完成率 ≥ 95%
4. **通知到达率**：微信服务号消息推送成功率 ≥ 90%（含降级处理后；部分用户已完成 OpenID 绑定，上线后持续推进宣发扩大覆盖率）
5. **系统可用性**：核心功能（用户录入、审核工作台）在外部 API 不可用时可降级运行
6. **转化漏斗基线建立**：系统上线后首月完成咨询→录入→入群→付费各环节数据采集，形成基线，后续以此设定量化提升目标

## 范围界定

**首版 MVP 包含：**
- 9个功能模块全量交付（Dashboard、用户管理、入群审核、消息通知、SLA预警、飞书好友管理、客服/导师/门派管理、权限控制）
- 4路外部 API 集成与降级策略
- 完整操作日志体系
- 角色权限矩阵（PRD 阶段细化）
- **历史数据迁移**：将现有 Excel 数据直接导表作为源数据导入系统，完成存量用户数据的历史衔接
- **上线支持**：内部培训 + 后台侧操作文档与核心名词注释，降低团队上手门槛

**首版 MVP 不包含：**
- 移动端 App
- 用户自助 H5 端（仅接入微信服务号绑定和通知推送）
- 高级报表导出 / 数据分析功能
- 与其他外部平台的新增 API 集成

**待 PRD 阶段明确：**
- 企微二维码文件大小上限
- 权限角色矩阵细节（各角色 × 各模块的菜单/操作/字段权限）
- 飞书 API / 第三方导师系统 API 不可用时的完整降级方案

## 未来演进方向

客户服务中心首版解决"数据分散、流程低效"的生存级问题。随着右豹平台规模扩大，系统有望演进为：

- **智能工单分配**：基于客服负载和用户特征自动分配工单，减少人工调度
- **用户生命周期追踪**：从首次咨询到付费、晋级、流失的全程数据追踪与干预提示
- **运营数据中台**：在转化漏斗基线建立后，逐步扩展为支持多维运营决策的数据分析平台，驱动业务增长策略
