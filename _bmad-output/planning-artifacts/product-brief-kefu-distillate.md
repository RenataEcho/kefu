## 一、技术选型（已确认）

- **架构**：低代码 + 定制后端混合
- **后端**：Node.js 或 Python/FastAPI，负责所有外部 API 中转、SLA 定时任务、消息通知队列与重试
- **数据库**：MySQL 主库 + Redis 缓存（外部 API 同步数据，每小时定时刷新，支持手动触发）
- **前端**：Vue3 + 开源后台模板（如 Arco Design Pro）；审核工作台、通知失败队列等复杂页面定制开发
- **消息队列**：处理通知发送的异步重试（服务号失败 → 记录失败队列）

## 三、外部 API 依赖与降级策略（已确认）

| API 来源 | 用途 | 不可用时降级行为 |
|---|---|---|
| 右豹APP API | 右豹编码校验（模块2录入时） | 整批/单条均无法提交，等待恢复 |
| 右豹APP API | 近10天动作数据同步（模块5/入群审核表） | 展示旧缓存数据 + 页面提示"数据暂未更新" |
| 飞书 API | 获取进群申请信息（模块1入群审核） | 管理员后台手动导入申请数据，手动操作审核，SLA 以手动录入的申请时间为计时起点 |
| 第三方系统 API | 导师与学员绑定关系 + 项目数据 | 展示缓存数据 + 页面提示（学员数、项目数字段均降级） |
| 微信服务号 | 向用户推送审核结果通知 | 推送失败记入通知失败队列，客服通过企微/飞书线下联系用户；短信降级人工执行 |

## 四、完整数据结构

### 用户主档表（13字段）

| 字段 | 关键规则 |
|---|---|
| 右豹编码 | 核心身份字段，全局唯一不重复，录入时 API 校验真实性 |
| 右豹ID | 与右豹编码为不同字段，付费流程截图核对 |
| 是否付费学员 | 布尔值；默认未付费；付费记录录入后变更为已付费；付费记录删除后自动回退 |
| 飞书ID | 手动录入，暂无校验 |
| 飞书手机号 | 手动录入，需校验格式 |
| 飞书昵称 | 文本 |
| 右豹项目收益 | 支持点击查看明细（主从结构，关联收益明细从表） |
| 右豹总关键词数 | 来自右豹APP API 同步 |
| 右豹总订单数 | 来自右豹APP API 同步 |
| 微信 OpenID | 用户关注服务号后获取，与右豹编码关联存储；**不在后台管理界面展示**；用于消息推送 |
| 录入时间 | 系统自动生成时间戳 |
| 所属客服 | 聚合展示普通客服+专属顾问；手动填写可编辑；一个用户可记录多个客服；空一方显示"-" |
| 所属门派 | 根据所属导师自动查询归属门派填充，非手动；导师为空或导师未关联门派时显示"-" |
| 所属导师 | 展示导师昵称+绑定时间；来自第三方API，非本系统录入；API 返回导师不存在时报错不自动创建 |

### 入群审核表（12字段）

- 字段：右豹编码、飞书手机号、飞书昵称、近10天关键词申请数、近10天回填数量、近10天订单数、申请入群昵称、申请入群ID、申请状态（待审核/已通过/已拒绝）、申请原因、申请时间（SLA计时起点）、拒绝原因
- 生命周期：同一用户可有多条记录；被拒后重新提交新增记录，旧记录自动归档（状态保持已拒绝，不删除）

### 收益明细从表（5字段）
- 字段：项目名称、项目类型、项目订单数、关键词数量、累计已结算收益

### 付费记录操作日志表（10字段）
- 字段：右豹编码（不可修改）、飞书手机号（不可修改）、付费金额（可改）、付费时间（可改，年月日格式）、付费对接人（自由文本，可改）、操作类型（新增/修改/删除，系统写入）、操作人、操作时间、操作前数据JSON、操作后数据JSON

### 客服表（5字段）
- 字段：客服ID（自增主键）、客服名称（唯一非空）、客服类型（普通客服/专属顾问）、企微二维码（必填，jpg/png，文件大小上限PRD确认）、建联用户数（只读统计）
- 删除规则：关联用户数 > 0 时拦截

### 导师表（8字段）
- 字段：导师ID、导师名称、导师手机号（需格式校验）、所属门派（必填，单门派）、当前负责项目数量（第三方API，只读）、当前学员数量（第三方API，只读）、学员总收益（聚合用户主档表，只读）、数据更新时间
- 删除规则：关联用户数 > 0 时拦截

### 门派表（9字段）
- 字段：社群ID、门派名称、门派介绍、门派负责人（自由文本，不关联人员表）、门派导师数量（只读）、门派学员数量（只读）、门派学员总收益（只读，可查明细）、门派相关项目总收益（只读，可查明细）、数据更新时间（收益更新或基础信息编辑时触发）
- 删除规则：关联导师数或学员数 > 0 时拦截

### 飞书添加记录表（7字段）
- 字段：记录ID、右豹编码、操作人ID、发起时间、飞书申请状态（待接受/已接受/已拒绝/超时）、手动标记标志（布尔）、最后更新时间
- 注意：此表仅在模块5视图展示，不写入用户主档表

### 操作日志通用设计
- 涉及操作日志的业务表：用户主档表、付费记录表、入群审核表、客服表、导师表、门派表
- 统一字段：关联业务记录ID、操作类型、操作人、操作时间、操作前数据JSON、操作后数据JSON
- 每个业务表各自维护独立的日志表

## 五、各模块关键业务规则

### 模块0：Dashboard
- 管理员/负责人：全局汇总（用户概览、门派概览、导师概览、客服概览）
- 客服：仅看本人建联用户数
- 转化漏斗：咨询→录入→入群→付费（上线首月采集基线数据，后续设量化目标）
- 收益数据来自右豹API每小时同步；门派/导师数据来自第三方API每小时同步
- 支持日期范围筛选（本月/本季度/自定义区间）

### 模块1：用户数据录入与审核
- 非付费用户：客服直接在系统录入（非用户自填），录入后发飞书群二维码
- 付费用户：协助注册右豹APP，要求截图提供右豹编码+右豹ID（共2张），客服核对录入
- 审核工作台：列表页直接操作，待审核橙色/已超时红色（>3天）
- 通过：一键操作→系统更新状态→调飞书API批准入群→推送微信服务号通知
- 拒绝：弹窗填写拒绝原因→更新状态→推送通知（含H5修改指引）→失败进入失败队列
- 拒绝原因为空时H5展示默认文案：「审核未通过，详细原因请联系客服」
- 审核中不支持重复提交；被拒后可重新提交（新记录，SLA重置，旧记录归档）
- 飞书入群状态同步三种方式并行：定时轮询、飞书webhook回调、后台手动刷新

### 模块2：录入付费用户
- 支持Excel批量导入和单条手动新增
- 校验逻辑：右豹编码通过API校验真实性；已付费用户报错不可重复录入；负数金额拦截，0允许
- 批量导入重复编码：仅对重复条目报错，同批未重复条目正常导入
- 导入完成展示「成功X条/失败X条」，支持导出失败明细Excel
- 付费时间格式：YYYY-MM-DD
- 支持修改（金额/时间/对接人）和删除；删除后用户状态自动回退为未付费
- API不可用时：整批/单条均无法提交，等待恢复

### 模块3：消息通知
- 主渠道：微信服务号；降级：短信（人工执行）
- OpenID绑定：服务号配置「达人绑定」H5菜单，用户主动填写右豹编码完成绑定；部分用户已绑定，上线后持续宣发推广
- 通知场景4种：审核通过、审核被拒、SLA超时首次预警（审核员）、SLA超时二次催促（审核员）
- 短信格式：「您在【{群名称}】的入群申请已{通过/驳回}，详情请点击：{H5链接}」
- 通知失败记录菜单：字段（通知时间、右豹编码、通知场景、通知渠道、推送状态），客服可见，只读
- SLA预警记录菜单：字段（预警时间、审核记录ID、预警类型、发送状态），管理员/审核员可见，只读

### 模块4：SLA超时预警
- 时效：3自然日；截止为第3个自然日23:59（北京时间UTC+8）
- 计时起点：入群审核表的"申请时间"字段
- 超时后：向所有"是否为审核员=true"的账号发第一次微信服务号加急提醒
- 二次催促：首次预警后8小时内仍未处理，再发一次；此后不再催促
- 扫描条件：只扫描申请状态=待审核的记录；已归档/已拒绝/已通过不参与

### 模块5：飞书好友添加优先级管理
- 用户范围：仅展示申请状态为待审核或入群审核中的用户（非全量主档）
- 优先级：近10天内有关键词申请或信息回填动作的用户优先；否则暂缓
- 一键发起：以具体客服/导师账号身份调用飞书API；前提是账号已完成飞书OAuth授权
- 离职/停用时需及时撤销授权
- 状态追踪：飞书API回调自动更新；支持操作人手动标记"我已添加"
- 数据同步：每天定时更新（非实时）；API不可用时展示旧数据+页面提示

### 模块6：权限控制
- 三维度：菜单权限 + 操作权限（新增/修改/删除/导入/导出）+ 字段显隐（如付费金额等敏感字段）
- 审核员：账号属性标记（"是否为审核员"），非独立角色；所有标记为审核员的账号均收SLA/审核通知
- 账号管理流程：管理员创建角色组→分配权限→新增账号时选角色组+填写名称/账号/初始密码+勾选是否审核员
- 账号可自助修改昵称和登录密码
- **待PRD设计：各角色清单及对应的菜单/操作/字段权限矩阵**

### 模块7/8/9：客服/导师/门派管理
- 客服类型变更后同步更新关联用户主档表的所属客服聚合字段
- 企微二维码必填（新增时），支持修改，格式jpg/png，大小上限待PRD确认
- 导师所属门派必填，一个导师只能属于一个门派
- 门派侧不支持分配/移除导师，导师归属在导师管理侧维护
- 门派负责人为自由文本，不关联系统人员表
- 三个模块统一删除规则：关联数>0时拦截，提示先解除关联

## 九、已明确拒绝的方案（不重复提议）
